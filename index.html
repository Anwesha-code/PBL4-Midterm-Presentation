Energy Consumption Prediction at Data Centres Using LSTM-RNN Optimised by Ninja Optimisation Algorithm

A multi-horizon forecasting framework using a Bidirectional stacked LSTM-RNN with NiOA-based hyperparameter optimisation.

Introduction

Data centres are amongst the fastest-growing contributors to global energy consumption and carbon emissions. Continuous computational workloads generate high-dimensional power demand signals. Accurate forecasting of energy increments — the additional energy consumed over a future interval — enables proactive scheduling, cooling optimisation, and intelligent resource management.

This project proposes a reproducible deep learning model for multi-horizon energy increment prediction. The core model is a Bidirectional Deep Recurrent Neural Network (DRNN) using LSTM whose hyperparameters are optimised by the Ninja Optimisation Algorithm (NiOA) — a population-based meta-heuristic inspired by the adaptive hunting strategies of ninjas, balancing exploration and exploitation across iterative refinement cycles.

Problem Statement

Accurate real-time prediction of energy increments remains challenging due to: high-dimensional sensor data, complex temporal dynamics, recording gaps of up to 65.8 days, extreme target distribution skewness (std ≈ 7–8× the mean), and difficulty of hyperparameter tuning at scale. Manual tuning is inefficient and sub-optimal.

Objectives

  • Design a Bidirectional DRNN architecture for multi-horizon cumulative energy increment prediction.
  • Implement gap-aware target computation to handle recording interruptions of up to 65.8 days.
  • Optimise hyperparameters automatically using NiOA.
  • Apply log1p variance-stabilising transformation and Huber loss to address extreme target skewness.
  • Evaluate across multiple forecast horizons: k = 60 s, 300 s, 900 s, and 1800 s.
  • Benchmark against Classical ML (LR, SVR, XGBoost, MLP), Deep Learning (Vanilla LSTM, CNN-LSTM), ARIMA, and DRNN+Optuna.

Proposed Methodology

Dataset

  • Source: IEEEDataPort — Data Server Energy Consumption (Estrada et al., SEIT 2022)
  • Duration: August–December 2021 (~5 months)
  • Raw rows: 3,178,051 at 1-second resolution
  • Max gap: 5,683,031 s (65.8 days)
  • Features (17): Voltage, Current, Power, Energy, Frequency, PF, Sensor Temp, CPU %, CPU Power, CPU Temp, GPU, GPU Power, GPU Temp, RAM, RAM Power, Hour, Weekday
  • Target: ΔEₖ(t) = E(t+k) − E(t)
  • Horizons: k = 1, 60, 300, 900, 1800 s

Pipeline Flow — 10 Stages

1

Load CSV with fault tolerance

Corrupt lines skipped; Spanish column names mapped to English equivalents.

2

Timestamp parsing & chronological sort

Forward-fill imputation; duplicate removal; strict temporal ordering.

3

Gap-aware target computation

Targets rejected where elapsed_time > 2k seconds or ΔE < 0 (counter reset).

4

Z-score outlier removal (features only)

|z| ≥ 3 removed from feature columns; target column excluded.

5

Target capping + log1p transform

99.5th percentile cap; np.log1p applied → near-Gaussian range [0, ~6.04]. Inverse np.expm1 at evaluation time restores kWh units.

6

Chronological 70/15/15 split

Strictly time-ordered. No shuffling. Verified by timestamp boundary assertions.

7

StandardScaler (train-only fit)

Fit on training split only; transform applied to val and test — prevents leakage.

8

Sliding window sequence generation

Window = 120 timesteps × 17 features → X shape (N, 120, 17). Stride tricks for memory efficiency.

9

NiOA hyperparameter search (30% subset)

6 agents × 7 evaluations = 42 function evaluations. Trial time limit: 1,800 s.

10

Final training on full training split

Huber loss (δ=1.0), AdamW, EarlyStopping (patience=7), batch=16. Best val epoch weights restored.

Model Architecture — Bidirectional Stacked DRNN

Input ( 120 × 17 )

→ Bidirectional LSTM ( units, return_sequences=True )

→ [ LSTM ( units ) + Dropout ] × (layers − 1)

→ BatchNormalization

→ GlobalMaxPooling1D

→ Dense ( 64, ReLU ) → Dropout

→ Dense ( 25, ReLU ) → Dense ( 1 )

  • Bidirectional: captures both forward and backward temporal patterns
  • Huber loss δ=1.0: robust to residual outliers in log1p space
  • AdamW: weight-decay regularisation decoupled from gradient
  • GlobalMaxPooling1D: aggregates most salient temporal features

v1 Critical Bug

energy.shift(-k) shifts by row index, not real time. Across 65.8-day gaps, ΔE = total gap energy (~417 kWh), not a 1-second increment → 240,000× train/val MSE gap.

v2 Remaining Issue

Positive counter-reset artefacts (meter register wraps to large positive value) pass the gap and negative-delta filters. ~10–17% of test samples contain ~417 kWh spurious values, dominating MAE.

v3 Improvements

Log1p transform + Huber loss eliminate MSE collapse on skewed targets. NiOA time limit 420→1800 s allows 4–6 epochs per trial for meaningful comparison.

GPU Memory Engineering — SequenceDataGenerator

X_train: (1,874,617 × 120 × 17) ≈ 15 GB. Both model.fit(X) and tf.data.from_tensor_slices() copy the full array to GPU → OOM. Solution: tf.keras.utils.Sequence subclass keeps 15 GB in CPU RAM; transfers only one batch (~0.1 MB) per GPU step. Peak VRAM = 1 batch regardless of dataset size.

Why Classical Models Are So Similar

All classical models produce near-identical MAE (~72.77) and RMSE (~174.19) because the same ~16% artefact samples (ΔE ≈ 417 kWh) dominate all metrics equally. Any model predicting near-zero incurs the same ~66 kWh error on artefact samples. The small differences reflect model quality on the 84% clean samples.

Discussion

What Worked Correctly

  • Gap-aware target computation eliminated the catastrophic 240,000× train/val MSE discrepancy of v1.0.
  • SequenceDataGenerator enabled training a 15 GB dataset on a consumer RTX 3060 GPU (12 GB VRAM) without OOM errors.
  • NiOA search converged and identified stable hyperparameters — consistent selection of 64-unit BiLSTM across all horizons.
  • Log1p + Huber loss eliminated MSE model collapse; training converged to meaningful validation losses (~6–8×10⁻⁶).
  • NiOA-DRNN outperforms all 4 classical ML benchmarks on the full test set (MAE 67.14 vs 72.77 kWh).
  • Clean-subset sMAPE improves from 96.2% (k=60) to 34.4% (k=900), confirming the model extracts genuine temporal patterns.
  • Strict chronological splitting with timestamp boundary assertions ensured zero data leakage.
  • All configurations centralised in config.py with fixed seeds — fully reproducible pipeline.

Identified Limitations & Remediation

Positive Artefact Spikes — Root Cause

Meter counter-reset events produce large positive ΔE (ΔE ≈ 417 kWh). They are temporally close (elapsed < 2k) and positive (not caught by negative-delta filter). ~10–17% of test samples contain these. They dominate MAE — the model is not failing, it correctly predicts normal values of ~0.005 kWh; the artefact samples incur ~417 kWh errors each.

Fix (Immediate Future Work)

Add physical plausibility filter before sequence generation: remove any row where ΔEₖ > (1000 W × k) / 3600 kWh. For k=60: threshold = 16.67 kWh. For k=300: 83.33 kWh. This single line of code will eliminate the artefact problem entirely and is expected to push clean-subset metrics to full-test-set metrics.

Outcomes & Achievements

  • Implemented GPU-safe large-dataset training via SequenceDataGenerator (15 GB on 12 GB VRAM).
  • Validated log1p + Huber loss combination for heavy-tailed regression targets at scale.
  • Successfully executed NiOA HPO with 42 evaluations across a 6-dimensional space on real data.
  • Completed full NiOA-DRNN training for k=60, 300, 900 with all artefacts saved and reproducible.
